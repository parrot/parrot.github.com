Iterates the elements of %hash in no apparent order, but the order will be the same between successive calls to these functions, as long as %hash doesn't change.If @keytests are provided, only elements whose keys evaluate $key ~~ any(@keytests) as true are iterated.What is returned at each element of the iteration varies with function. keys only returns the key; values the value; kv returns both as a 2 element list in (key, value) order, pairs a Pair(key, value).Note that kv %hash returns the same as zip(keys %hash; values %hash)In Scalar context, they all return the count of elements that would have been iterated.The lvalue form of keys is not longer supported. Use the .buckets property instead.
