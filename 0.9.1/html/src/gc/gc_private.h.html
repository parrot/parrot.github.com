Astute observers may notice that if the PObj is comprised of only an INTVAL, then there are some systems where sizeof(PObj*) can be larger then sizeof(PObj), thus creating overflow. However PObjs are never used by themselves, things like PMCs and STRINGs are cast to PObj in the GC, so we should have plenty of space. */ typedef union GC_MS_PObj_Wrapper { PObj obj; PObj *next_ptr; } GC_MS_PObj_Wrapper;
